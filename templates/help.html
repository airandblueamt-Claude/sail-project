How It Works

What SAIL Is For

SAIL is the AMT control team's tool for logging issues against company assets, assigning repair work to operation-team members, and building a per-asset issue history over time. End users do not log in to SAIL — they email the control team, and the team logs the ticket on their behalf.

Who Uses SAIL

  • Control-team admins / managers — raise tickets, assign work, manage equipment and staff records.
  • Operation-team technicians — receive ticket assignments by email, log in to update status, add comments, and write the resolution.

Daily Workflow

  1. An end user emails the control team about a broken or faulty asset.
  2. Open SAIL → Equipment Catalog or Manage Assets → find and click the affected asset.
  3. Click Raise New Issue → fill in the issue category, priority, title, description, and the affected user's name and email address.
  4. Submit. The affected user receives an email confirming the issue has been logged.
  5. Open the new ticket → set Assigned To to the operation-team member who will fix it.
  6. The assigned technician receives an email notification. They log in to SAIL, update the ticket status, add progress comments, and write a resolution note when the work is done.
  7. When the technician sets the status to Resolved (with a resolution note), the affected user is automatically notified that their issue has been fixed.

Adding an Operation-Team Member

  1. Go to Employees in the sidebar.
  2. Click + Add Employee → enter the person's name and email address. Role defaults to Technician.
  3. Submit. Tell the new member their temporary password is Aramco@123 — they will be prompted to change it on first login.
  4. To reset a forgotten password, open the employee's edit page and click Reset password.

Adding a New Issue Category

Go to Issue Categories in the sidebar → type the new category name → click Add. Use this whenever a new failure mode appears that doesn't fit an existing category (e.g. "Network port fault", "Screen damage").

Asset Issue History

Every asset has its own detail page that lists every ticket ever raised against it, including the resolution. This is the team's institutional memory — use it to spot repeat failures or patterns before escalating to procurement.

Navigate there via Equipment Catalog → model → asset tag, or directly from Manage Assets.

Reports

The Inventory Report (/reports/inventory) and Ticket Report (/reports/tickets) show weekly and monthly rollups with CSV export. Available to admins, managers, and technicians.

Email Status

The dashboard shows a green dot when outbound email is active, or a grey dot when it is not configured. If grey, set the SAIL_SMTP_PASSWORD environment variable to a Gmail app password and restart the server.
